# 프로젝트 작업 계획표

## 1. 개요
- 본 문서는 PCINS Payment Gateway API 문서화 프로젝트의 유지보수 및 작업 진행을 위한 계획표입니다.
- 새로운 작업을 진행할 때 본 계획표를 먼저 업데이트하고 진행합니다.

## 2. 작업 항목
- [x] **Step 1:** 프로젝트 전체 구조 분석 (`openapi.yaml`, `package.json` 등 파악 완료)
- [x] **Step 2:** 프로젝트 전체 요약 마크다운 작성 (`PROJECT_SUMMARY.md` 생성)
- [x] **Step 3:** Apidog API 문서와 현재 스펙 비교 분석 (`API_SPEC_DIFF_REPORT.md` 생성)
- [x] **Step 4:** `openapi.yaml` 전면 수정 (Apidog 현행화, 테스트 서버 제거, Redoc 우측 응답 예시 강화)
- [x] **Step 5:** 스펙 Lint 검증 확인 및 문서 사이트 화면(Redoc) 디자인 최적화 검토
